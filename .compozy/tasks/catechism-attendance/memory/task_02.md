## Objective Snapshot

## Important Decisions

- **RLS helpers as functions**: Used `is_coordinator()` and `is_class_catechist(p_class_id)` SQL functions (SECURITY DEFINER) instead of inline subqueries in every policy — keeps policies readable and consistent.
- **UNIQUE constraints placement**: `UNIQUE (class_id, date)` on `attendance_sessions`; `UNIQUE (session_id, student_id)` on `attendance_records` — required for idempotent offline sync upsert.
- **Trigger uses COALESCE with defaults**: `handle_new_user` defaults role to 'catechist' if `raw_user_meta_data->>'role'` is null — safe fallback.
- **TypeScript types in `lib/supabase/types.ts`**: Added `isValidRole`, `isCoordinator`, `isCatechist`, `ALL_TABLES` runtime helpers (not just type aliases) so they appear in coverage.
- **Tests are SQL-structure tests + mocked RLS**: Real DB integration tests require Docker daemon or real Supabase credentials — neither available. Tests validate SQL file contents and mock Supabase client behavior.
- **`supabase init` auto-creates `config.toml`** but NOT the `migrations/` directory — must `mkdir -p supabase/migrations` manually.
